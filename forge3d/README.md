# NovaForge 3D

**Изкована от филамент** — статична витрина/каталог за 3D печатни изделия (маски, флейти, вази, лампи и др.), отпечатани на **Bambu Lab H2S**. Собственик: **Yasin (@Bedeaznam)**, България.

Сайтът е чист HTML/CSS/JS без build стъпка — работи директно на **GitHub Pages**. Поддържа **4 езика** (BG / EN / RU / UK) със смяна от клиента и **потребителски акаунти** чрез Firebase Authentication.

> ⚠️ Това е **витрина** с профили и ръчни поръчки. За пълноценен онлайн магазин с **онлайн плащания** ще е нужен допълнителен backend/платежен доставчик (напр. **Stripe**, Shopify, Snipcart). Виж [Плащания](#плащания).

## Структура на проекта

```
index.html                  Началната страница (каталог, за нас, профил) — една страница със секции
assets/
  ├── background.svg         Фоново изображение (blur слой .bg + тъмен .shade слой)
  ├── i18n.js                Всички преводи (BG/EN/RU/UK) + логика за смяна на език
  ├── app.js                 Каталог с продукти + навигация
  ├── auth.js                Firebase Authentication (регистрация/вход/изход/профил)
  ├── firebase-config.js     ⬅️ ТУК попълваш конфигурацията на твоя Firebase проект
  └── product-*.svg          Placeholder снимки на продуктите
README.md
```

Дизайнът е в **glassmorphism** стил (полупрозрачни карти с `backdrop-filter: blur`, заоблени ъгли 26px, chip бутони с pill форма), с неонова оранжево/лилава тема за мейкър/техно бранд.

## Как да добавяш / редактираш продукти

Всички продукти са в масива `PRODUCTS` в **`assets/app.js`**. Всеки продукт изглежда така:

```js
{
  id: "mask-aztec",                 // уникален идентификатор
  img: "assets/product-mask.svg",   // път до снимката (виж по-долу)
  price: "€39",                     // цена (показва се както е записана)
  name: { bg: "Маска", en: "Mask", ru: "Маска", uk: "Маска" },
  desc: { bg: "Описание...", en: "Description...", ru: "...", uk: "..." }
}
```

- **Добавяне** на нов продукт: копирай един блок `{ ... }`, смени `id`, `img`, `price` и текстовете за 4-те езика.
- **Изтриване**: махни съответния блок.
- Заглавието и описанието **задължително** имат 4 езика (`bg`, `en`, `ru`, `uk`), за да работи преводът.

## Как да качваш снимки

1. Сложи снимката (JPG/PNG/WebP/SVG) в папка **`assets/`**, напр. `assets/my-mask.jpg`.
2. В `assets/app.js` посочи пътя в полето `img`, напр. `img: "assets/my-mask.jpg"`.
3. Препоръчително съотношение **4:3** (напр. 800×600 px) за еднакъв изглед на картите.

## Как да добавяш / редактираш преводи

Всички текстове са в обекта `translations` в **`assets/i18n.js`**, разделени по език (`bg`, `en`, `ru`, `uk`). Всеки текст има ключ, напр.:

```js
bg: { "about.title": "За нас", ... },
en: { "about.title": "About us", ... },
```

- За да **смениш текст**, редактирай стойността за съответния ключ във всичките 4 езика.
- За да **добавиш нов текст в HTML**, сложи атрибут `data-i18n="твой.ключ"` върху елемента (или `data-i18n-ph="твой.ключ"` за placeholder на input), после добави ключа в 4-те езика в `i18n.js`.
- Езикът по подразбиране е **BG**; изборът се пази в `localStorage` (`nf_lang`) и се помни при следващо посещение.

## Настройка на Firebase Authentication (безплатно)

Регистрацията и входът използват **Firebase Authentication**. Firebase има безплатен план (Spark), достатъчен за старт. Стъпки:

1. Влез в **[Firebase Console](https://console.firebase.google.com/)** с Google акаунт и натисни **Add project** (Добави проект). Дай име (напр. `novaforge-3d`) и завърши създаването.
2. В левия панел отвори **Build → Authentication → Get started**.
3. В таба **Sign-in method** активирай:
   - **Email/Password** → Enable → Save.
   - (по желание) **Google** → Enable → избери support email → Save.
4. Върни се на **Project settings** (иконата ⚙️ горе вляво) → секция **Your apps** → натисни иконата за уеб **`</>`**.
5. Дай му псевдоним (напр. `web`) и **Register app** (не е нужно да включваш Hosting).
6. Ще видиш блок **`firebaseConfig`** с реални стойности. Копирай ги.
7. Отвори **`assets/firebase-config.js`** в този проект и замени placeholder стойностите (`YOUR_API_KEY` и т.н.) с твоите:

   ```js
   export const firebaseConfig = {
     apiKey: "AIza...........",
     authDomain: "novaforge-3d.firebaseapp.com",
     projectId: "novaforge-3d",
     storageBucket: "novaforge-3d.appspot.com",
     messagingSenderId: "1234567890",
     appId: "1:1234567890:web:abcdef..."
   };
   ```

8. **Важно (Authorized domains):** във Firebase → Authentication → **Settings → Authorized domains** добави домейна, от който ще се хоства сайтът, напр. `bedeaznam.github.io` (и твоя custom домейн, ако имаш). `localhost` вече е разрешен за тестове.
9. Запази, commit-ни и push-ни промените. Готово — формите за регистрация/вход ще заработят.

> Докато конфигурацията не е попълнена, формите остават видими, но показват съобщение „не е конфигуриран“ вместо да опитват вход. Така сайтът работи веднага, дори без Firebase.

⚠️ **Firebase web `apiKey` не е таен** — нормално е да е публичен в клиентски код. Достъпът се пази чрез Authorized domains и правилата за сигурност, не чрез скриване на ключа.

## Публикуване на GitHub Pages

1. Push-ни съдържанието в GitHub repository (напр. `Bedeaznam/forge3d`).
2. В repository-то: **Settings → Pages**.
3. **Source:** *Deploy from a branch* → Branch: `main` (или `master`) → folder `/ (root)` → **Save**.
4. След минута сайтът е достъпен на `https://<username>.github.io/<repo>/` (напр. `https://bedeaznam.github.io/forge3d/`).

### Custom домейн (CNAME)

Ако купиш домейн (напр. `novaforge3d.com`):

1. Създай файл **`CNAME`** в корена на repo-то, съдържащ **само** домейна:
   ```
   novaforge3d.com
   ```
   *(Все още не е създаден нарочно — добави го, когато имаш реален домейн.)*
2. При регистратора на домейна добави DNS записи към GitHub Pages:
   - `A` записи за apex домейна към GitHub IP адресите (185.199.108–111.153), **или** `CNAME` запис за поддомейн (`www`) към `<username>.github.io`.
3. В **Settings → Pages → Custom domain** въведи домейна и включи **Enforce HTTPS**.
4. Добави домейна и във Firebase → Authentication → Authorized domains.

## Плащания

Настоящият сайт е **витрина** — клиентите разглеждат каталога, създават профил и се свързват със собственика за ръчна поръчка (бутонът „Поръчай“ в момента показва съобщение — може да се насочи към имейл/форма).

За **реални онлайн плащания** е нужен допълнителен слой, който GitHub Pages сам по себе си не осигурява (защото е само статичен хостинг):
- **Stripe** (Checkout / Payment Links) — най-бърз старт; Payment Links дори не изискват backend.
- **Snipcart** / **Shopify Buy Button** — вграждат количка в статичен сайт.
- Собствен backend (Node/Serverless функции) за обработка на поръчки и плащания.
